HashLink target

  • A virtual machine for Haxe
  • Announced in October last year
  • Available in Haxe 3.4 in November
  • Used to develop and ship Northgard (production stable!)

HL Dual compilation

  • Bytecode:
    • Compile with haxe -hl myapp.hl ...
    • Run with hl myapp.hl <args>
    • JIT x86 (not yet x86_64)
    • Similar to Neko + FASTER
    • Really faster!
  • Native code:
    • Generate with haxe -hl out/myapp.c ...
    • Compile with gcc -o myapp -std=c11 out/myapp.c -Lhl
    • x86 / x86_64 / ARM?

HL Dual compilation

  • Shared runtime
    • Standard library
    • Garbage collector
    • Value in-memory representation
  • Full C compatilibity
  • Very low native dependencies
  • Windows / OSX / Linux / Mobile

HashLink Future

  • Consoles : PS4 / Nintendo Switch / Xbox / etc.
  • Debugger / VSCode integration
    • (Windows only at first)
  • x86_64 JIT, better for Linux/OSX daily dev
  • better GC performances
  • Want to learn more?
    • HashLink + Heaps.io workshop (Thur/15:05)
