Haxe: future - Haxe 4.0

Haxe 4

  • Haxe is mature (language wise)
  • Priority : tooling/quality - over language feature Y
  • 1.897 commits since last year conference

Haxe latest features

  • Lua/PHP7 targets improvements
    • new targets introduced in 3.3/3.4
    • more stable, better code output, bugfixes
  • New "Eval" macro interpreter
    • 4x+ speed up !
  • Compilation Server
    • Cache the code, allows fast compilation/completion
    • more stability, bugfixes

Haxe tools support

  • Visual Studio Code integration
    • vshaxe extension
    • Credit to IDE team (Dan, Jens, Simon)
  • Compiler Services:
    • Completion
    • Hover type tips
    • Refactoring (rename var, find all refs...)
    • Diagnostics (realtime errors, unused var/import, no side effect...)
  • Goal: standard IDE for Haxe
    • Alternative: IntelliJ
    • Most of the tools can integrate into other IDEs

More Haxe 4

  • Arrow functions: arr.map(x -> x + 1)
  • New function type syntax: (x:Int,y:String) -> Int
  • New final keyword: final x = 0;
  • Final for fields: typedef Point = { final x : Int; final y : Int; }
  • Syntax lift: @:enum abstract becomes enum abstract etc.

Plans for Haxe 4+

  • Unicode by default (mostly affect Neko and C++ targets)
  • for( key => value in map ) iteration
  • inline XML markup for DSLs/React/etc.
  • async / await implementation (all platforms)
  • ... and more (contribute on github/haxefoundation/haxe-evolution)

HashLink: today

HashLink target

  • A virtual machine for Haxe
  • Announced in October 2016 last year
  • Made available in Haxe 3.4 in November
  • HashLink 1.5 compatible with Haxe 4.0-RC3
  • Used to develop and ship Northgard (production stable!)

HL Dual compilation

  • Bytecode:
    • Compile with haxe -hl myapp.hl ...
    • Run with hl myapp.hl <args>
    • JIT x86 and x86_64
    • Similar to Neko + FASTER
    • Really faster!
  • Native code:
    • Generate with haxe -hl out/myapp.c ...
    • Compile with gcc -o myapp -std=c11 out/myapp.c -Lhl

HL Dual compilation

  • Shared runtime
    • Standard library
    • Garbage collector
    • Value in-memory representation
  • Full C compatilibity
  • Very low native dependencies
  • Windows / OSX / Linux / Mobile / Consoles (!)
  • VSCode integration with full Debugger support!
  • Want to learn more? [Heaps.io Sat.3pm]
