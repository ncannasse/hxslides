Langage Haxe

  • Basé sur des classes
  • Mélange OO/fonctionnel
  • PAS pur OO:
  • class, interface, typedef, structure, enum, abstract
  • Syntaxe classique (Java/JS/C-like)

Langage Haxe

  • Inférence de type
  • 		function foo( k ) {
			var str = "Hello World !";
			var a = [];
			for( i in 0...k )
				for( word in str.split(" ") )
					a.push(word);
			return a;
		}
		
  • 		function foo( k : Int ) : Array<String>
  • Le compilateur est un allié, pas un ennemi

Langage Haxe

  • Le tout-puissant Dynamic
  • 		var x : Dynamic = ...;
		// fait ce que l'on veut avec x
		// responsabilité du développeur
  • Accés direct et sans filet à la plateforme
  • Utiliser l'API Reflect pour du multi-plateforme

Langage Haxe

  • Sous typage structurel
  • 		typedef AnyPoint = { x : Float, y : Float };

		function distance( a : AnyPoint, b : AnyPoint ) {
			var dx = a.x - b.x;
			var dy = a.y - b.y;
			return Math.sqrt(dx * dx + dy * dy);
		}
		
  • 		distance( new Point(5,1), { x : 1.5, y : -1 });

Langage Haxe

				var g = Std.int(v*255);

			case Mix(a,b):

Macros FTW

  • Programme executé à la compilation
  • ...qui prends en paramètre du code
  • ...et retourne un autre programme
  • 			macro function getBuildDate() {
				var str = Date.now().toString();
				return { expr : EConst(CString(str)), pos : ... };
			}

Macros FTW

  • accèdent aux fichiers
  • accèdent à vos types
  • peuvent modifier l'AST
  • mais pas la syntaxe
